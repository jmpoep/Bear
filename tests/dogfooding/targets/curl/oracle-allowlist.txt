# Oracle allow-list for the curl target (dogfood-divergence-report).
#
# This file suppresses KNOWN, benign argument-level differences on translation
# units that are matched on (file, normalized-output) between Bear's capture
# and CMake's own compile_commands.json. It is deliberately small and is NOT
# used to paper over coverage gaps: TUs present in only one database are the
# "extras" report (logged, never failing), not allow-list material.
#
# How it is applied (run.sh oracle branch): for every "differing" entry the
# comparator reports, each rule below is applied to BOTH sides' argument lists
# (drop the matching tokens from a_arguments and from b_arguments alike). If,
# after applying every rule, the two argument lists are equal, the entry is a
# benign difference and is suppressed. The gate PASSES iff no differing entry
# survives. Because rules are applied symmetrically to both sides, a rule can
# only ever hide a difference that is exactly the allow-listed pattern; a real
# extra flag on one side alone survives and fails the gate.
#
# Rule grammar (one rule per non-comment, non-blank line):
#   flag <TOKEN>           drop every occurrence of the literal <TOKEN>
#   flag-with-arg <TOKEN>  drop <TOKEN> and the single argument token that
#                          immediately follows it (the value it consumes)
#
# ---------------------------------------------------------------------------
# Why each entry is benign
# ---------------------------------------------------------------------------
#
# Depfile generation flags. CMake's exported compile_commands.json records the
# *configure-time* compile line, which omits the make/ninja dependency-tracking
# flags; the real build rule that Bear intercepts adds them. They affect only
# the .d dependency side-file the build system consumes, never the produced
# object. On this curl build the ENTIRE matched-but-differing set (221/221 TUs)
# differs ONLY by this flag group; with these three rules the differing set is
# empty. The consumed arguments are the object path (-MT) and the depfile path
# (-MF); pairing the flag with its argument keeps the rule from silently
# dropping an unrelated token.
flag -MD
flag-with-arg -MT
flag-with-arg -MF
